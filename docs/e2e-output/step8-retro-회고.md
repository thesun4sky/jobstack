## 2. 무엇이 잘 되었는가 (Keep)

### K1. 솔직함이 자산이다
- Q3(실패 경험)에서 운영 DB 사고를 솔직하게 인정 → 진정성 5/5 최고점
- Q4(지식 공유)에서 "인턴이라 주도적 경험이 없다"는 솔직한 인정 → 진정성 5/5
- **교훈**: 진정성은 면접관의 호감을 얻는 가장 강력한 무기. 이 강점을 유지하되, 솔직함 위에 **구체성**을 더해야 함

### K2. 실전 에피소드가 있다
- Redis 캐싱, 슬로우 쿼리 최적화, 운영 DB 사고, 밥친구 프로젝트 — 소재 자체는 충분
- **교훈**: 에피소드는 있으니 **풀어내는 방식(STAR 구조 + 수치)**만 개선하면 됨

### K3. 기술 직무 연결 시도
- Q2에서 "성능 최적화에 관심→검색 플랫폼" 연결 시도
- **교훈**: 방향은 맞으나, 연결 근거가 약함 → 기업 연구로 보강 필요

## 3. 무엇이 문제였는가 (Problem)

### P1. 수치 없는 답변 — 구체성 2.8/5
**증상**: "좋은 결과", "많이 빨라졌다", "뿌듯했다"
**근본 원인**: 인턴/프로젝트 당시의 정량적 성과를 사전에 정리하지 않음
**영향**: 면접관이 "이 사람은 성과를 측정하는 습관이 없구나"라고 판단할 위험

### P2. 논리 구조 미완성 — 논리성 2.8/5
**증상**: "상황 설명 → 결과" 직행. "왜 그 선택을 했는지"가 빠져 있음
- Q1: "투표로 결정" → 왜 투표가 최선이었는지 근거 없음
- Q2: "Redis를 적용해보겠다고 자원" → 왜 Redis인지 근거 없음
- Q5: "기술력이 뛰어나서" → 다른 회사도 기술력 뛰어남

**근본 원인**: STAR(Situation-Task-Action-Result) 중 Task와 Action 부분이 약함

### P3. 희망형 종결 반복
**증상**: 면접 답변에서도 "~싶습니다", "~것 같습니다" 빈출
- Q1: "잘 마무리되었습니다" (수동적)
- Q2: "뿌듯했습니다" (감정형)
- Q4: "참여하고 싶습니다" (희망형)
- Q5: "성장하고 싶습니다" (희망형)

**근본 원인**: 자소서 작성 습관이 면접 답변에도 그대로 전이됨

### P4. 미끼 포인트 미활용 — 활용률 20%
**증상**: 자소서에 심어둔 미끼 5개 중 1개만 부분 활용
**근본 원인**: 자소서 수정본의 핵심 수치/키워드를 면접 답변용으로 별도 연습하지 않음

### P5. 기업 연구 부족
**증상**: Q5에서 "가장 많이 사용하는 서비스", "기술력이 뛰어나다" — 범용적 답변
**근본 원인**: 네이버 D2, DEVIEW 등 기술 콘텐츠를 구체적으로 소비하지 않음
