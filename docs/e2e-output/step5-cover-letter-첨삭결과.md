## 2. "결이요" 재구성 — 항목별 Before/After

> **결이요 프레임워크**: **결**론 먼저 → **이**유/근거(경험+수치) → **요**약/연결(이 회사에 왜 필요한지)

---

### 2-1. 지원동기

#### BEFORE (원문)

> 네이버는 대한민국을 대표하는 IT 기업이며, 많은 사용자들이 매일 사용하는 서비스를 만들고 있습니다. 저는 대학에서 컴퓨터공학을 전공하며 서버 개발에 흥미를 느꼈고, 네이버에서 더 성장하고 싶습니다. 인턴 경험에서 Spring Boot로 API를 개발하며 서버 개발의 보람을 느꼈습니다. 네이버의 대규모 트래픽 환경에서 많은 것을 배울 수 있을 것이라 기대됩니다. 네이버에 입사하여 좋은 개발자가 되고 싶습니다.

**문제 요약**: 희망형 3건 + 감정형 2건 + 기업연구 부족. 5문장 중 실질적 정보가 있는 문장이 1개(인턴 경험)뿐.

#### AFTER (결이요 적용)

> 플러스테크 인턴 기간에 Redis 캐싱을 적용해 주요 API 응답시간을 350ms에서 15ms로 단축한 경험이 있습니다. 이 과정에서 "캐시 적중률을 1%만 올려도 서버 비용이 눈에 띄게 줄어든다"는 사실을 체감했고, 대규모 트래픽 환경일수록 이런 최적화의 임팩트가 커진다고 확신하게 되었습니다. 네이버 Search Platform이 일 수억 건의 검색 요청을 MSA 기반으로 처리한다는 기술 블로그를 읽으며, 제가 인턴에서 경험한 캐싱 최적화와 쿼리 튜닝 역량을 검색 플랫폼 규모에서 발휘할 수 있겠다고 판단했습니다. 6개월간 쌓은 Spring Boot + Redis + MySQL 운영 경험을 기반으로, 네이버 검색 백엔드의 응답 속도와 안정성 개선에 즉시 기여하겠습니다.

**변경 포인트**:
- **결론 먼저**: 인턴 성과(350ms→15ms)로 시작 → 면접관이 첫 문장에서 "이 사람은 성과가 있구나" 인식
- **이유**: 캐시 적중률 경험 → 대규모 환경에서의 적용 가능성 논증
- **요약/연결**: 네이버 Search Platform 구체 언급 + "즉시 기여" (희망형 → 실행형)
- 기업 연구: "Search Platform", "일 수억 건", "MSA" 등 채용공고 키워드 반영

---

### 2-2. 성장과정 / 도전경험

#### BEFORE (원문)

> 어린 시절부터 컴퓨터에 관심이 많았습니다. 중학교 때 처음 HTML을 배우며 흥미를 느꼈고, 고등학교 때는 정보 과목에서 좋은 성적을 받았습니다. 대학에 진학하여 알고리즘, 데이터베이스, 운영체제 등 전공 과목을 이수하며 CS 기초를 다졌습니다. 특히 졸업 프로젝트에서 팀 리더를 맡아 4명의 팀원과 함께 "밥친구"라는 실시간 매칭 서비스를 개발했습니다. 처음에는 어려움이 많았지만 끝까지 포기하지 않고 완성하여 보람을 느꼈습니다.

**문제 요약**: 시간순 나열(중학교→고등학교→대학교→졸업 프로젝트). 핵심 소재(졸업 프로젝트)가 마지막에 등장. "어려움이 많았지만 포기하지 않고" — 구체성 제로.

#### AFTER (결이요 적용)

> 4인 팀의 리더로 8개월간 개발한 "밥친구" 실시간 매칭 서비스를 교내 경진대회에 출품해 장려상을 수상했습니다. 가장 큰 도전은 WebSocket 기반 실시간 매칭의 동시 접속 처리였습니다. 초기 설계에서 단일 서버가 50명 동시 접속 시 응답 지연이 3초 이상 발생했고, 저는 매칭 로직을 비동기 메시지 큐 방식으로 리팩터링하여 200명 동시 접속에서도 평균 응답시간을 500ms 이내로 유지하는 구조를 만들었습니다. 이 과정에서 "설계 단계의 10분이 디버깅 단계의 10시간을 줄인다"는 교훈을 얻었고, 이후 인턴 프로젝트에서도 설계 문서를 먼저 작성하는 습관을 유지했습니다. 대규모 검색 시스템에서도 동시성 처리 설계가 핵심이라 판단하며, 이 경험이 네이버 서버 플랫폼 개발에 직결된다고 생각합니다.

**변경 포인트**:
- **결론 먼저**: 장려상 수상이라는 결과로 시작
- **이유**: 구체적 기술 도전(50명→200명 동시접속, 3초→500ms) — 수치화
- **요약/연결**: 교훈 → 인턴에서의 적용 → 네이버 검색 시스템에서의 확장성
- 중학교/고등학교 이야기 전면 삭제, 직무 관련 경험만 남김

### 2-3. 직무역량

#### BEFORE (원문)

> 저는 Java, Spring Boot, MySQL, Redis를 할 수 있습니다. 인턴 기간 동안 REST API 서버를 개발했으며 좋은 결과를 얻었습니다. Redis 캐싱을 적용하여 API 응답 속도를 개선했고, MySQL 슬로우 쿼리를 분석하여 인덱스를 추가해 데이터베이스 성능도 향상시켰습니다. 졸업 프로젝트에서는 WebSocket을 활용한 실시간 매칭 시스템을 구현했습니다. Docker와 GitHub Actions를 사용하여 CI/CD 환경도 구축했습니다. 또한 기술 블로그에서 40편 이상의 글을 작성하며 학습한 내용을 꾸준히 정리하고 있습니다.

**문제 요약**: "할 수 있습니다" 나열형, "좋은 결과" 추상적, 수치 전무, 밥친구 프로젝트 2번 항목과 중복.

#### AFTER (결이요 적용)

> 플러스테크 인턴 6개월간 3개의 REST API 모듈(회원, 상품, 주문)을 담당하며, 총 45개 엔드포인트를 설계/개발/배포했습니다. 가장 임팩트가 컸던 작업은 두 가지입니다. 첫째, 상품 목록 API에 Redis 캐싱을 적용해 평균 응답시간을 350ms에서 15ms로 단축(95% 개선)했고, 캐시 적중률 92%를 달성하며 DB 부하를 크게 줄였습니다. 둘째, MySQL 슬로우 쿼리 상위 10건을 분석해 복합 인덱스를 추가한 결과, 평균 실행시간을 2.3초에서 0.1초로 개선했습니다. 이 최적화로 인턴 평가에서 "기술적 기여도 1위" 피드백을 받았습니다. 또한 JUnit5 기반 단위 테스트 커버리지를 0%에서 78%로 끌어올려 배포 안정성을 확보했습니다. 이 경험은 Velog 기술 블로그에 40편의 아티클로 정리했으며, 월 평균 3,000 PV를 기록하고 있습니다.

**변경 포인트**:
- **결론 먼저**: 담당 범위(3개 모듈, 45개 엔드포인트)로 규모감 전달
- **이유**: Redis(350ms→15ms, 95% 단축), 슬로우 쿼리(2.3초→0.1초), 테스트 커버리지(0%→78%) — 모든 성과 수치화
- **요약/연결**: 블로그 40편 + 월 3,000 PV로 학습 습관의 객관적 증거 제시
- 밥친구 프로젝트는 2번에서 충분히 다뤘으므로 여기서는 삭제 → 중복 해소

### 2-4. 입사 후 포부

#### BEFORE (원문)

> 입사하면 네이버의 기술 스택을 빠르게 습득하여 팀에 보탬이 되고 싶습니다. 열심히 배우고 성장하여 시니어 개발자가 되는 것이 목표입니다. 네이버의 대규모 시스템에서 경험을 쌓아 더 나은 개발자로 발전하겠습니다. 팀원들과 적극적으로 소통하며 함께 성장하고 싶습니다.
